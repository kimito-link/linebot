# 実装ハンドオフ — 販売LP「自由に人格を作れるLINE AIボット」

> この1枚だけで着手できる。設計の背景・判断根拠は [`LP-PERSONA-AI-DESIGN.md`](./LP-PERSONA-AI-DESIGN.md) を参照（読む必要が生じたときだけ）。
> 実装は別モデル/次チャットで行う。このドキュメント作成時点では未着手。

## スコープ（MVPだけ）

`apps/lp/index.html` を全面書き換える。現行版（「あなただけの人格と、LINEで話せる。」ヒーロー＋persona-gridの版）を、設計書の第B部セクション構成に置き換える。

**やること**
- 設計書 第B部の9セクション構成でHTMLを組む
- 設計書 第C部のコピーをそのまま使う（微調整は可、方向性は変えない）
- 設計書 第D部のチャットモック仕様を実装（タブ切替、JS 30行以内）
- FAQは`<details>/<summary>`でJS不要のアコーディオンにする
- 既存アセット4枚（`apps/lp/assets/characters/{link,link-normal,konta,tanunee}.png`）は§8最終CTAの案内役としてのみ使用。チャットモック内アバターには使わない（設計書F部・地雷7）

**やらないこと（今回のMVPスコープ外）**
- ブランド名「ツクモ」の正式決定（下記「着手前に必ずやること」参照。未検証のまま確定名として印刷物・ドメイン取得はしない）
- デモ用LINE公式アカウントの用意（体験直結CTAは将来対応、今回はフォームCTAで可）
- privacy/tokushohoページの中身作成（フッターリンクは残すが遷移先ページは別タスク）

## 着手前に必ずやること（実装より前・法務/事実確認）

これらは設計書が「地雷」として明示した、コピーを書く前提条件。飛ばして実装すると虚偽表示や規約違反のリスクがある。

1. **ブランド名の可用性チェック**（設計書 第0部A）: 「ツクモ」で商標検索(J-PlatPat)・Google/X検索・LINE内アカウント名検索・ドメイン確認。衝突があれば同軸の次候補へ。
2. **プライバシー実装の事実確認**（設計書 地雷5）: 会話ログの保存範囲・第三者提供の有無・AIプロバイダ側の学習利用設定を確認し、「ふたりの会話は、ふたりだけのもの」の注記文言を実態に合わせる。`apps/worker/src/services/`のログ関連実装（`llm-reply.ts`, `groq-pipeline.ts`等）を読んで裏取りする。
3. **LINE公式アカウントのカテゴリ審査要件確認**（設計書 地雷2）: 「夜の話し相手」を出会い系・アダルトカテゴリと誤認されない文言か、申請前に確認。

## 実装手順

1. ブランチを切る（例: `feat/lp-persona-rewrite`）
2. `apps/lp/index.html` を設計書 第B部の構成で書き直す。CSSは現行ファイルのウォームトーン（ピンク×バイオレット）から、設計書第D部の配色（桜/ラベンダー/ネイビー/オレンジのアクセント×ウォームオフホワイト基調）へ更新
3. ヒーローのタブ切替を実装（設計書第D部「切替JS」参照。データ属性`data-persona`、`hidden`属性の付け替え、CSSトランジション）
4. §4マルチモーダル表現、§2シーンカード6枚（故人の一文込み）、§5安心セクション、§6料金、§7 FAQ、§8最終CTA、§9フッターを順に実装
5. ローカルで `npx serve apps/lp` 等で目視確認（前回同様、file://直開きは画像パス解決の都合で避ける）
6. コミット→ユーザー確認→pushとVercel `--prod`デプロイの可否を確認してから実行（前回同様、明示許可があるまで本番反映しない。ただし「本番にすぐ上げていい」という既存の合意がある場合はその範囲に従う）

## 機械的な完了判定

- [ ] `apps/lp/index.html`が設計書第B部の9セクションを全て含む
- [ ] GitHubリンク・技術スタック言及（Cloudflare Workers/Groq/Gemini等）・「FOR DEVELOPERS」セクションが本文から削除されている
- [ ] チャットモックにLINE緑(#06C755系)・「既読」表示・LINE型ヘッダーが**存在しない**（grepで`06C755`と`既読`が本文に出ないことを確認）
- [ ] 全チャットモックに「※画面は本サービス独自のイメージです」のキャプションがある
- [ ] 故人の一文が設計書 第C部の文言のまま（誇張・効能語を追加していない）
- [ ] タブ切替がJS無効時も1タブ目が表示される（`hidden`属性のフォールバック構造）
- [ ] FAQが`<details>/<summary>`実装（追加JS不要）
- [ ] ローカルサーバーで全アセット・全セクションがHTTP 200で疎通確認済み

## 地雷（設計書からの転記・実装時に踏みやすい順）

1. LINE公式ロゴ・スクショの使用禁止（ブランドガイドライン抵触）
2. 「甘い夜」「大人の」等の性的示唆語彙を使わない
3. 故人の一文に効能語（癒される/立ち直れる等）を足さない
4. 「必ず」「誰でも」等の断定効能コピーを使わない、会話例に「イメージ」注記必須
5. 「もう独りじゃない」「ずっと一緒」系の依存助長コピーを使わない
6. チャットモック内アバターにりんく達の画像を使わない（商品人格との混同）
